Overview

Overview

During my third year I participated in a group project in a team of 8. Over the course of 8 months we created a 2D game conforming to a brief an external client. Our client FuturLab collaborated by providing direction and feedback over the course of the project.
The team included:

  • Me as the engine programmer,
  • 1 gameplay programmer,
  • 1 production manager,
  • 1 game designer,
  • 3 artists,
  • 1 sound engineer.
A video of the prototype is available. The recording quality is not ideal as it had to be recorded using an external camera.

My Role - Engine Programmer

As the engine programmer my job was to structure the application and provide the gameplay programmer with the tools necessary to focus on gameplay. The work I accomplished during my time on the project includes:

  • Managing the public GitHub repository for the project. Building the application will not be possible as the Abertay Framework is not included and a PSVita devkit is required.
  • Finite state machine for the overall application structure.
  • AppState interface for creating application states, the IntroState, the MenuState and the LevelState (base class for in-game states).
  • The tiling system, including the map loading funtionality. Levels are loaded in using RapidXML to parse a .tmx file.
  • The physics bodies and the collision response for tiles using Box2D.
  • All menus and the in-game user interface.
  • The camera functionality.

Concurrent A* Pathfinding

A* pathfinding implementation multi-threaded using the C++11 threading library to calculate many paths concurrently.

Concurrent A* Pathfinding

  • Back
  • C++
  • Download .zip (232KB)

Overview

  • Implementation of A* pathfinding using Manhattan heuristic.
  • Task-based farming of pathfinding done using C++11 threading library.
  • Completed paths are outputted through a channel using a condition variable and a mutex.
More details are available in the presentation for the project - available in the download.

Download

The download contains two Windows executables, the project presentation and the performance report. The first is the example shown on the right which calculates the shown path. The second randomizes the board (sometimes resulting in impossible paths).

Example

The example shown calculates the same path, depicted on the left of the image, 50 times. Diagonal movement is allowed and weighted equal to orthogonal movement.

  • ! is the start point. ? is the goal.
  • # is an obstacle.
  • $ is a cell on the best path.
  • - is a rejected cell.
  • + is a cell in the working set (candidate for being checked).
  • To prove the channel works, the first completed path outputs the keys of all cells on the path to the goal - shown on the right.

Gas Giant

The mesh is an ico-sphere generated recursively from an icosahedron. The sphere has a 1D texture mapped vertically which is manipulated in the pixel shader using GPU generated simplex noise.

Gas Giant

  • Back
  • C++
  • Download Report (6.18MB)

Overview

  • Recursively generated icosphere mesh using midpoint displacement, the sphere is 1D texture mapped.
  • GPU generated Simplex Noise used to deform the 1D texture, visually approximating a gas giant in real time.
  • Gaussian blur and screen space texture distortion post processing effects.
