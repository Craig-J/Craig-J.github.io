Overview

Augmented Reality Game

A two-stage game involving a shape-matching puzzle stage and a battle stage where you avoid and shoot down incoming shapes.

Augmented Reality Game

  • PSVita
  • C++
  • 3D Graphics
  • Back
  • Download Report (860KB)

Overview

  • Two staged augmented reality game implemented using the Abertay Framework for Sony PSVita.
  • The first stage is a shape-matching puzzle. The player shape is on AR marker 1, the target shape is on AR marker 2. The player changes their shape by moving the markers. Three transforms are used to determine the players shape. The type of the players shape is changed by rotating marker 1 - as depicted on the controls image. The shape can be either a sphere or cube. The scale of the players shape is changed based on the distance between the markers. The colour of the palyers shape is changed based on the relative angle of marker 2 around marker 1 - visualizable as a colour wheel centered on marker 1. The defined colours are shown on the controls image.
    Once the shapes match (same type and colour, reasonably similar scale) the second stage can be entered by pressing Cross, Square or Triangle to select the difficulty level.
  • The second stage is combat where shapes will fire at the player from marker 1. The player can shoot down these shapes with their own by firing using the shoulders buttons and can move around to avoid incoming shapes by moving the PSVita. To win, the player must destroy a variable number of enemy shapes depending on the difficulty level. If the player is hit by an incoming shape they lose.

Networked Space Game

Generic Space Game was originally implemented on the PSVita using the Abertay Framework. I then ported it to SFML for desktop compatibility and implemented networking capabilities using WinSock.

Networked Space Game

  • Windows/PSVita
  • C++
  • 2D Graphics
  • Networking
  • Back
  • Download .zip (4.88MB)

Overview

  • Infinite side scrolling space game where the player must avoid or destroy oncoming asteroids and enemy spaceships.
  • Score is awarded for destroying asteroids or enemies - the player has a cannon and photon weapon to destroy enemies with.
  • The game acts as a server and sends messages to any connected spectator clients. Spectator clients recieve messages for all input and events (such as enemy spawns) and reproduce the game deterministically. TCP protocol is used and messages are executed on a frame basis (not time) delayed by a few hundred frames so that the spectator clients have a large buffer to catch up after packet loss.

Download

The download contains three main items (not including readme):

  • Server executable - this is the playable game executable.
  • Client executable - will automatically connect to server (local machine only) to recieve game-state messages.
  • Project presentation - a short presentation about the networking features of the applications.
There are several known issues for the application, explained in the readme.

Gas Giant
