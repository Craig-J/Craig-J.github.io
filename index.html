Galaxea - Professional Group Project

Overview

During my third year at university I participated in a group project in a team of 8.

Over the course of 8 months we created a 2D game conforming to a brief an external client.

Our client, FuturLab, collaborated by providing direction and feedback over the course of the project.

The team included

  • Me as the engine programmer,
  • 1 gameplay programmer,
  • 1 production manager,
  • 1 game designer,
  • 3 artists,
  • 1 sound engineer.

A video showcasing a complete playthrough of the game is available below. The video quality is not ideal due to recording on an external camera.

My Role

As the engine programmer my job was to structure the application and provide the gameplay programmer with the tools necessary to focus on gameplay.

I managed the Git resository for the project. The repository is publicly viewable here. Building the application will not be possible as the Abertay Framework is not included and a PSVita devkit is required.

The work I accomplished during my time on the project includes:

  • The overall application structure including the state machine.
    I created the image below near the start of the project to illustrate the original design for the application structure to the team. The final structure differed a little from this image. LevelSelect became a part of MenuState and the ScoreScreen was simply an overlay shown during the level at the end rather than a whole state.
  • All code pertaining to the AppState base class, the IntroState, the MenuState and the majority of the LevelState base class was written by me.
  • The tiling system, including the loading funtionality, the physics bodies using Box2D and the collision response. Levels are loaded in using RapidXML to parse a .tmx file.
  • All menus and the in-game user interface.
  • The camera functionality.
1D Texture Manipulated Icosphere

A textured ico-sphere mesh generated recursively from an icosahedron.
The sphere has a 1D texture mapped vertically which is manipulated using GPU generated simplex noise.

Threaded Pathfinding

A* pathfinding implementation originally written as coursework for the Pathfinding module.
The pathfinding algorithm was then threaded using the C++11 threading library to take calculate many paths concurrently.
